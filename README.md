# 🛡️ Crisis Navigation System (CrisisNav)

**CrisisNav** is an industry-grade, smart emergency guidance platform designed to transform complex, paper-based Standard Operating Procedures (SOPs) into clear, actionable, step-by-step digital workflows. 

Built for enterprises, safety teams, and emergency response units, CrisisNav ensures that when seconds count, precision and compliance are never compromised.

---

## 🚀 Features

### 💎 Premium Industrial UI
- **Modern SaaS Aesthetic:** Inspired by Palantir, Linear, and Stripe dashboards.
- **Glassmorphism Design:** Sophisticated UI using blur and saturation effects for a premium feel.
- **Dark/Light Hybrid Theme:** Optimized for high-stakes environments with deep navy and industrial blue accents.

### 🧭 Smart Emergency Navigator
- **Step-by-Step Guidance:** Converts dense SOP manuals into digestible interactive steps.
- **Automated Action Logs:** Automatically records timestamps and completed actions for post-incident audit and compliance.
- **Severity Classification:** Intelligent routing of protocols based on the intensity of the crisis (Low, Medium, High, Critical).

### 📊 Industry Command Center
- **Real-time Monitoring:** A mock dashboard reflecting facility status, active alerts, and SOP compliance metrics.
- **Site-Wide Visibility:** Designed for multi-site enterprise management.

### 🛡️ Future AI Capabilities
- **Crisis Detection:** NLP engine to detect emergencies from communications and sensor feeds.
- **Auto-Scanner:** Rapidly digitize legacy paper SOPs using computer vision.
- **Decision Support:** AI-driven recommendations based on historical incident data.

---

## 🛠️ Technology Stack

- **Core:** Vanilla JavaScript (ES6+), HTML5 Semantic Structure
- **Styling:** Vanilla CSS3 (Custom Variables, CSS Grid, Flexbox, Glassmorphism)
- **Icons:** Phosphor Icons Library
- **Architecture:** Lightweight static-hosting ready, compatible with FastAPI/PostgreSQL backends.

## �🏃 How to Run Locally

The project is designed to be lightweight and requires zero installation for the frontend prototype.

1. **Clone the repository** (or ensure you have the project files).
2. **Start a local static server**. You can use Python as it's built-in to most systems:
   ```bash
   python -m http.server 8000
   ```
3. **Open your browser** and navigate to:
   `http://localhost:8000`
